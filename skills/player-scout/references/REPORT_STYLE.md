# Report Style Reference — player-scout

Visual conventions for the single-player scouting `.docx`. Load only when generating the report.

## Palette (aligned with usta-team-scout)

| Name | Hex | Usage |
|------|-----|-------|
| Navy | `1F3864` | Section headings, header cell fill |
| Light blue | `D6E4F0` | Key/value label cells |
| Win green | `E2EFDA` | W cell shading in match log |
| Loss red | `FCE4D6` | L cell shading in match log |
| Warning red | `C00000` | Self-rated warning callout |

## Cell and row conventions

| Element | Convention |
|---------|-----------|
| Section headings | Navy, bold |
| Key/value header cells | Navy fill, white text |
| Key/value label cells | Light blue fill |
| Match log — win row | W cell shaded green |
| Match log — loss row | L cell shaded red |
| Self-rated warning paragraph | Bold, dark red |
| Rating trend — rising | Green label |
| Rating trend — falling | Red label |
| Rating trend — steady | Neutral label |

## Match log columns

`Date | Type (S/D) | Court | Opponent Team | Opponent(s) + rating | Score | W/L | Post-match DR`

## Style goals

A captain reading the report should have an opinion about this player within 60 seconds. TL;DR bullets carry the weight; everything below is evidence. Keep the log compact — the full history is a reference, not a headline.
